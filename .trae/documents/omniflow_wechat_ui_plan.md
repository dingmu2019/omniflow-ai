# OmniFlow 顶级体验与微信级交互重构计划

## 1. 摘要
彻底重构 OmniFlow 的前端 UI 与交互逻辑，将其升级为**仿微信移动端的单列沉浸式应用**。实现真正的全模态响应式布局、多语言（中/英）、多主题（明/暗）切换，并引入“隐式无感连接”技术，使用户在未拨打电话时也能自然地发送图文，全面对标微信/ChatGPT 的行业顶级水准。

## 2. 当前状态分析
- **UI 布局**: 现为左右分屏的桌面端设计，响应式仅做简单折叠，缺乏移动端原生 App 的沉浸感。
- **功能入口**: 文字和文件输入框默认隐藏，必须先点击全局的“连接”按钮，交互逻辑与现代聊天软件（常驻输入框）严重脱节。
- **通话交互**: 缺乏原生的“拨号、接听、挂断”全屏/悬浮通话界面。
- **国际化与个性化**: 暂不支持多语言（i18n）切换和明暗主题（Dark/Light Mode）切换。

## 3. 具体修改方案

本次重构主要集中在 `src/static/index.html`，无需修改后端核心引擎。

### 3.1 微信级单列沉浸式布局
- **整体框架**: 采用 `max-width: 500px`、高度 `100vh` 的居中容器（在移动端自动 100% 撑满屏幕）。
- **常驻输入区**: 底部固定类似微信的输入栏，包含文本输入框、表情按钮（可选）、扩展菜单 `+` 按钮。
- **扩展菜单**: 点击 `+` 弹出面板，包含“相册 (发图)”、“语音通话”、“视频通话”等入口。
- **聊天气泡**: 升级为微信经典的“用户绿色气泡、AI 白色/灰色气泡”，并增加时间戳和头像占位。

### 3.2 独立原生通话面板 (Call Overlay)
- 新增一个全屏覆盖的 `#callOverlay` 层。
- **触发机制**: 用户在 `+` 菜单中点击“语音/视频通话”时弹出。
- **界面元素**: 
  - 顶部：最小化按钮（退回聊天界面但不挂断）。
  - 中部：视频流区域或 AI 语音呼吸动效（能量球），以及底部的沉浸式电影字幕。
  - 底部控制栏：静音/取消静音、开关摄像头、红色的“挂断”按钮。

### 3.3 隐式无感连接架构 (Implicit Connection)
- 重构 WebRTC 初始化逻辑。
- 如果用户在**未通话状态**下直接发送文本或图片，前端会在后台静默调用 `/connect` 接口建立房间，以**仅开启 Data Channel（不请求麦克风权限）**的方式加入房间。
- 随后将文本/图片发送给后端 Agent 触发回复。
- 当用户后续点击“语音通话”时，系统只需在现有的连接上 `setLocalAudio(true)` 并请求权限，实现秒级热启动升级。

### 3.4 多语言与多主题支持
- **CSS 变量 (多主题)**: 在 `:root` 定义 `--bg-color`, `--text-color`, `--chat-bg` 等变量，通过为 `<body>` 切换 `.light-theme` 类来实现明暗切换。
- **前端 i18n 字典 (多语言)**: 建立 JS 静态文本映射表（ZH/EN），遍历带有 `data-i18n` 属性的 DOM 元素进行文本替换。
- **快捷入口**: 在顶部导航栏提供显眼的 🌓 (主题) 和 🌐 (语言) 快捷按钮。

## 4. 假设与前置决策
- **纯前端改造**: 依赖于后端的 Daily Transport 已经支持 Data Channel，因此隐式文本聊天无需后端改动，仅靠前端 WebRTC 状态机控制即可。
- **轻量化**: 不引入 Vue/React 或 i18next 等重量级框架，纯原生 HTML/CSS/JS 实现，保持极致的加载速度和可移植性。

## 5. 验证步骤
1. **视觉验证**: 界面变为单列布局，底部输入框常驻。点击 🌓 按钮，界面平滑切换为浅色/深色；点击 🌐，界面文案在中英之间切换。
2. **纯文本验证**: 刷新页面后，不点击任何通话按钮，直接输入文本发送。检查控制台网络请求，确认系统在后台建立连接并能收到 AI 的文本回复。
3. **原生通话验证**: 点击 `+` -> 语音通话，确认弹出全屏通话界面，AI 能够通过语音回复，且字幕正常显示。点击挂断按钮，通话界面收起且 WebRTC 连接正确断开或降级。
